# Release Manager Skill

Use this skill when a feature, milestone, demo, or project version is ready to hand off.

## Step 1: Define release target

```md
## Release Target
- Version / milestone:
- Audience: users / teacher / judges / teammates / client / developers
- Target deliverable:
- Release date:
- Included PRs / tasks:
```

## Step 2: Summarize user-facing changes

```md
## User-facing Changes
| Change | User value | Related requirement | Evidence |
|---|---|---|---|
```

## Step 3: Summarize technical changes

```md
## Technical Changes
| Area | Change | Risk | Notes |
|---|---|---|---|
```

## Step 4: Check release readiness

```md
## Release Readiness Checklist
- [ ] P0 requirements completed.
- [ ] Acceptance criteria verified.
- [ ] Main demo flow verified.
- [ ] Known bugs documented.
- [ ] Rollback or recovery plan documented.
- [ ] Required screenshots / videos / docs prepared.
- [ ] README or usage instructions updated if needed.
```

## Step 5: Known issues

```md
## Known Issues
| Issue | Impact | Workaround | Follow-up task |
|---|---|---|---|
```

## Step 6: Release notes

Produce two versions:

### Internal release notes
Detailed for teammates and developers.

### External release notes
Clear for users, teachers, judges, or clients.

## Step 7: Goal Alignment Check

```md
## Goal Alignment Check
- Original target deliverable:
- Release package produced:
- Match status: Match / Partial / Off-track
- Missing or risky items:
- Next correction:
```
